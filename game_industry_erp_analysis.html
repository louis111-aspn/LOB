🎯 핵심 명제
게임업계는 무형자산 중심의 IP 비즈니스이며, 수익인식·원가배분·내부거래 모두 IP 라이프사이클에 종속됩니다. ERP 설계는 IP 단위 수익/원가 추적 가능성(Traceability)을 최우선으로 해야 합니다.

법인 유형별 핵심 비즈니스 프로세스
SAP FI/CO 관점에서 각 법인에 필요한 프로세스와 구성 요소
CO — 프로젝트 원가 관리
  • PS (Project System) 또는 Internal Order 활용
  • 게임 타이틀 = WBS Element 최상위 단위
  • 단계(기획/개발/베타/서비스) = 하위 WBS
  • 인건비·외주비·서버비 WBS 집계
  • 자산화 전환: AuC(건설중자산) → 무형자산 (IM 모듈)
FI — 무형자산 회계
  • IAS 38: 연구단계 즉시비용화, 개발단계 요건 충족 시 자산화
  • Asset Class: 게임소프트웨어(5~10년 상각)
  • 손상검토: 타이틀별 현금창출단위(CGU) 지정
  • 조기종료 시 즉시 상각/손상 처리
1
프로젝트 개시 → WBS/IO 생성
PM Office에서 타이틀 코드 확정 후 SAP PS에 WBS 생성. 예산(Budget) 설정 및 Availability Control 활성화. 인력 배정은 HR 모듈 Cost Center에서 Activity Type을 통해 WBS로 집계.
2
개발비 집계 → 자산화 전환 판단
매월 말 CFO/회계팀이 자산화 요건 충족 여부 검토. 요건 미충족 시 P&L 비용화. 충족 시 AuC 계정으로 이관. 단계 완료 시 AuC → 무형자산 Settlement.
3
출시 후 — 라이브서비스 원가 성격 변화
게임 출시(Gold) 이후 발생하는 서버 운영비, 패치 개발비의 자본화 여부 재판단. 기능 추가(DLC/대규모 업데이트) vs 유지보수 구분이 핵심.
⚡ IFRS 15 핵심 이슈 — 퍼블리셔
게임 퍼블리셔는 다수의 수행의무(PO)가 하나의 계약에 혼재. 거래가격 배분이 비정형적이며 가변대가(환불, 할인, 로열티) 처리 자동화 필수.

📐 설계 원칙
게임 트랜잭션 볼륨은 초당 수천 건 이상 가능. SAP에 실시간 연동하는 것은 부적합 — 게임 DB에서 집계 후 주기적 배치(1시간~1일)로 SAP에 연동하는 구조 권고. 단, 환불·차지백 등 이벤트성 처리는 준실시간 처리 설계.

⚠ 게임 그룹사 IC의 특수성
게임 IP는 무형자산이므로 IP 이전·사용·로열티 등이 IC 거래의 핵심. IP의 이전(판매)인지, 라이선스(사용권)인지에 따라 연결 조정 방식이 전혀 다름. IP 내부 이전 시 미실현이익은 자산에서 제거되어야 하며, 이는 상각 기간 동안 지속적으로 추적 필요.

주요 리스크 및 ERP 구축 전략
게임 그룹사 SAP 프로젝트에서 반복 발생하는 실패 패턴과 대응 전략
HIGH RISK
게임 트랜잭션 볼륨과 SAP 성능
온라인 게임의 경우 초당 수천~수만 건의 아이템 구매 트랜잭션 발생. SAP에 직접 연동 시 DB 성능 한계 도달. 미들레이어 집계 설계 없이 Go-Live 시 장애 발생 가능.
✓ 대응 전략
게임 DB에서 시간 단위 집계 후 배치 인터페이스. SAP S/4HANA의 HANA In-Memory 활용으로 조회 성능 확보. 실시간 KPI는 SAC Live Connection.
HIGH RISK
IFRS 15 수행의무 식별 오류
배틀패스·시즌패스 등 번들 상품의 수행의무 식별 실패 시 매출 인식 시기 오류. 특히 콘텐츠 업데이트 의무가 내재된 경우 기간 이연 필요 여부 판단 오류 빈번.
✓ 대응 전략
상품 유형별 수행의무 식별 정책 사전 문서화. SAP Revenue Accounting & Reporting(RAR) 모듈 도입 검토. 법무·회계팀과 협업하여 아이템 코드별 매핑 테이블 관리.
MED RISK
개발비 자산화 기준 불일치
법인별, 프로젝트별 자산화 기준이 통일되지 않을 경우 그룹 연결 시 비교 가능성 저하. 특히 인수된 스튜디오가 기존 정책과 다른 기준 적용 시 조정 필요.
✓ 대응 전략
그룹 공통 개발비 자산화 정책 수립 및 SAP WBS 상태 코드와 연동. Alpha/Beta/Gold 단계 코드로 자동화 트리거 설계.
MED RISK
다통화 IC 거래 미매칭
개발사(KRW)와 글로벌 퍼블리셔(USD) 간 IC 로열티 거래에서 환율 적용 기준 불일치로 연결 시 IC 차이 발생. 월말 환율 vs 평균환율 적용 기준을 사전 확정해야 함.
✓ 대응 전략
그룹 IC 환율 정책 문서화. SAP GR에서 IC 파트너 통화 자동 변환 설정. 월별 IC Reconciliation 프로세스 의무화.
LOW RISK
SAC 데이터 신뢰성 (게임 Analytics 통합)
게임 운영 데이터와 SAP 재무 데이터의 정의 불일치(매출 집계 시점, 환불 처리 방식 등)로 SAC 대시보드 신뢰도 저하 발생 가능.
✓ 대응 전략
단일 진실의 원천(SSOT) 정책 수립. SAP 재무 데이터가 최종 기준. 게임 Analytics KPI는 보조 레이어로 명확히 구분.
