Section 01

제약·바이오 산업 구조와 밸류체인

완제·원료·바이오·CMO 유형 분류와 R&D→허가→생산→유통의 전체 흐름을 이해한다

🏭제약·바이오 기업의 4가지 유형

ERP 구축 범위와 프로세스 복잡도는 기업 유형에 따라 근본적으로 달라집니다. 첫 번째 질문은 항상 "이 회사가 어떤 유형인가?"여야 합니다.

💊완제의약품사 (Finished Dose)

원료를 구매하여 정제·캡슐·주사제 등 최종 제형으로 제조·판매. 영업/마케팅·허가관리·유통 프로세스가 핵심.

예시 한미약품, 유한양행, 종근당

ERP 범위: SD(영업), MM(구매), PP-PI(생산), QM(품질), FI/CO(원가)

🧪원료의약품사 (API Manufacturer)

Active Pharmaceutical Ingredient(API) 합성·제조. B2B 판매 중심으로 배치 추적성(Traceability)과 CoA(성적서) 관리가 핵심.

예시 일동제약 API부문, 글로벌 CDMO 일부

ERP 범위: PP-PI, QM, MM, 수출통관 (GTS)

🔬바이오의약품사 (Biopharmaceutical)

세포·단백질·항체 기반 바이오로직스 제조. 배양 공정 조건 관리, 역가(Potency) 시험, Cold Chain이 제조 원가의 핵심 변수.

예시 삼성바이오로직스, 셀트리온

ERP 범위: PP-PI + MES 연계 필수, LIMS 연동 복잡

🤝CMO/CDMO (위·수탁 제조)

의뢰사 제품을 위탁 생산. 다품목 다고객 Batch 분리, 고객사 BOM/레시피 격리 관리가 핵심 과제.

예시 삼성바이오로직스(이중), 롯데바이오로직스

ERP 범위: 고객별 가격구조(SD), 원가센터 분리, 기밀성 권한 설계

🔄제약·바이오 밸류체인 전체 흐름

Section 02

GMP란 무엇인가 — 규제가 ERP를 지배한다

GMP 핵심 원칙을 이해하고, ERP가 왜 GMP Compliant해야 하는지 논리적으로 설명할 수 있다

⚠️ 왜 제약 ERP가 어려운가
일반 제조업 ERP는 "생산을 효율적으로 관리"가 목적입니다. 제약 ERP는 여기에 "생산의 모든 과정을 법적으로 입증"이라는 조건이 추가됩니다. 이 차이가 설계 복잡도를 두 배 이상 높입니다.

🏛️GMP (Good Manufacturing Practice) 핵심 원칙

GMP는 의약품이 의도된 목적에 맞게, 일관된 품질로, 안전하게 제조되도록 보장하는 규제 체계입니다. ERP 관점에서 3가지 원칙이 가장 중요합니다.

📋Batch Record

모든 제조 활동의 완전한 기록. 누가, 언제, 어떤 원료로, 어떤 공정 조건에서 생산했는지를 증명. ERP에서 Process Order + PI Sheet로 구현.

🔍Audit Trail

모든 데이터 입력·수정·삭제의 이력 자동 기록. 사후 수정이 불가하거나 추적 가능해야 함. 21 CFR Part 11의 핵심 요건.

✍️Electronic Signature

종이 서명을 대체하는 검증된 전자서명. 단순 체크박스 불가 — ID/PW 재입력 또는 생체인증 수준의 인증이 필요.

🖥️CSV vs CSA — ERP 시스템 검증의 패러다임 전환

CSV (Computerized System Validation)

전통적 접근법. 시스템의 모든 기능을 사전 정의된 테스트 스크립트로 검증. 방대한 문서화 부담. SAP 표준 기능 변경 시마다 재검증 필요.

CSA (Computer Software Assurance)

FDA 2022 권고 기반의 현대적 접근. 리스크 기반(Risk-based)으로 검증 범위를 설정. 비즈니스 리스크가 높은 기능에 집중하여 효율화.

S/4HANA 구축 시 CSA 접근이 현실적으로 유리.

📊21 CFR Part 11 — ERP 설계 체크리스트

미국 FDA의 전자기록·전자서명 규정. 한국 식약처 KFDA에서도 유사 기준 적용.

💡 왜 SAP 표준 설정 변경이 규제 이슈가 되는가? ▼

CSV/CSA 관점에서 SAP 시스템은 "검증된 상태(Validated State)"로 유지되어야 합니다. 설정(Configuration) 또는 커스텀 개발(ABAP)을 변경하면, 해당 변경이 기존 검증을 무효화할 수 있습니다.

따라서 제약 고객사는 Change Control 절차를 통해 모든 시스템 변경을 관리합니다. ERP 구축 시 "이 변경이 Change Control 대상인가?"를 항상 Quality 팀과 협의해야 합니다.

컨설턴트 실무 팁: SAP 표준 기능의 Configuration 변경은 낮은 리스크, ABAP 커스텀 개발은 높은 리스크로 분류됩니다. 가능한 표준 기능 활용을 우선하는 이유가 규제 리스크 최소화에 있습니다.

Q. GMP 규제에서 Audit Trail이 요구하는 것은 무엇인가?
① 생산 수량과 원가만 기록하면 충분하다
② 최종 결과 데이터만 유지하면 된다
③ 데이터의 생성·수정·삭제 모든 이력을 자동으로 기록해야 한다
④ 품질 검사 결과만 전자서명으로 승인하면 된다
정답: ③ — Audit Trail은 원본 데이터가 아니라 모든 변경 이력을 추적합니다. 누가, 언제, 무엇을, 왜 변경했는지가 증명 가능해야 FDA 실사를 통과할 수 있습니다.
Section 03

제약 BOM의 본질 — 역가(Potency)와 수율

일반 BOM과 역가 BOM의 구조적 차이를 이해하고, 역산 로직과 수율 처리를 설명할 수 있다

📌 역가(Potency)란 무엇인가
역가(力價, Potency)는 원료 의약품이 함유하고 있는 유효성분(Active Ingredient)의 실제 함량 비율입니다. 같은 원료라도 로트(Lot)마다 역가가 다를 수 있으며, 이것이 일반 제조업 BOM과 근본적으로 다른 이유입니다.

⚙️일반 제조업 BOM

철판 2kg → 프레임 1개. 투입량이 고정입니다. 원자재의 "순도"나 "역가"를 고려할 필요가 없습니다.

Fixed BOM
투입량 = 명세서 고정값

🧬제약 역가 BOM

API 100mg 확보하려면? 역가 95%짜리 원료라면 105.26mg이 필요합니다. 역가 98%짜리라면 102.04mg이 필요합니다. 투입량이 로트마다 변동합니다.

Potency-Based BOM
투입량 = 목표 API 함량 ÷ 역가(%)

📐역가 BOM 계산 사례 — 단계별 이해

아목시실린 캡슐 250mg을 1배치(100,000캡슐) 생산한다고 가정합니다.

항목값비고
목표 API 함량250mg/캡슐 × 100,000 = 25,000,000mg (25kg)허가된 제조법 기준
원료 API 역가 (이번 로트)96.5%LIMS 시험 결과
실제 투입 원료량25kg ÷ 0.965 = 25.907kgERP 자동 역산
수율 (Yield) 80% 적용25.907kg ÷ 0.80 = 32.384kg 확보 필요공정 손실 반영
⚠️ ERP 구축 핵심 이슈
역가 값은 LIMS(실험실 정보 시스템)에서 관리됩니다. ERP가 이 값을 실시간으로 받아 BOM 투입량을 자동 계산해야 합니다. LIMS↔ERP 인터페이스 설계가 구축의 핵심 난제입니다.

🏗️3단계 BOM 구조 — 원료·반제품·완제품

제약 BOM은 단층이 아닙니다. 원료(API+부형제) → 반제품(Bulk) → 완제품(Finished Dose)의 3단계 구조를 가지며, 각 단계마다 역가와 수율이 적용됩니다.

💡 역가 기준(Potency Basis) — As-Is, 건조기준, 무수기준의 차이 ▼

As-Is (현재 상태 기준): 수분 포함 그대로의 실제 역가. 로트 수령 즉시 계량·재고 관리에 사용. 수분 함량에 따라 배치 간 편차 발생.

건조기준 (Dried Basis): LOD(건조감량) 보정 후 역가. 수분의 영향 제거. 규격서 작성에 주로 사용.

무수기준 (Anhydrous Basis): 수분 및 결정수 모두 제외. 항생제 등 결정수 변동이 큰 원료에 적용.

ERP 설계 시사점: 레시피와 LIMS 역가 보고의 기준이 일치해야 합니다. 레시피는 건조기준, LIMS는 As-Is로 보고한다면 시스템이 자동 변환 로직을 보유하거나 불일치를 명시적으로 표시해야 합니다. 이 설계 누락이 배치 과다/과소 투입 오류로 이어집니다.

Section 04

Batch 생산과 원가의 특수성

Batch 단위 원가계산 구조와 유효기간·반품 등 제약 특유의 원가 이슈를 설명할 수 있다

⚗️왜 "Batch"가 원가의 기본 단위인가

일반 제조업은 생산수량 × 단위원가로 원가를 산출합니다. 제약은 Batch(배치) 전체를 하나의 원가 객체로 관리합니다. 하나의 배치 안의 제품들은 동일 원가를 공유합니다.

이유: GMP에서 한 배치는 동일 조건에서 제조된 균질한 단위입니다. 배치 내 개별 단위를 별도 원가 객체로 분리하는 것은 규제상 의미도 없고 실무적으로도 불가합니다.

1 Batch = 1개의 원가 집계 단위
역가·수율 변동에 따라 원가도 배치마다 상이
불합격 배치 = 전액 폐기 또는 재작업 원가 추가

📦유효기간(Shelf Life)과 재고 관리의 특수성

제약 재고는 단순히 "있다/없다"가 아닙니다. 유효기간, 보관 조건, Batch 상태(합격/보류/불합격)가 재고 가용성을 결정합니다.

🗓️SLED (Shelf Life Expiration Date) 관리

SAP Batch Management + Shelf Life 설정으로 구현. 핵심 이슈:

• Standard MRP의 한계: 기본 MRP는 유효기간 만료 예정 배치를 가용재고로 간주. 계획 시점에 충분해 보이다가 실행 시점에 만료 배치가 되는 "Chaos" 발생

• 해결책: S/4HANA PPDS의 특성 기반 계획(Characteristics-Based Planning) 또는 IBP 연동

🌡️보관 조건과 재고 분류

동일 품목이라도 보관 조건에 따라 다른 재고로 관리해야 합니다:

• 상온 (15~25°C) / 냉장 (2~8°C) / 냉동 (-20°C 이하) / 초냉동 (-70°C)

• SAP Storage Location + Warehouse Management 설계 시 온도 구역 분리 필수

• Cold Chain 위반 이력이 있는 배치는 즉시 품질 보류 상태 전환

↩️반품·회수(Recall) — 제약 특유의 역물류

의약품 반품과 회수는 일반 제조업과 차원이 다른 규제 절차를 수반합니다.

유형발생 원인ERP 처리원가 영향
유효기간 만료 반품약국·병원의 기간 만료 재고Return Delivery → 폐기당기 손실 처리
자발적 회수 (Class I–III)품질 결함 또는 안전성 이슈Quality Notification → 리콜 오더회수 비용 + 재고 손실
행정 반품약가 인하, 품목 삭제Credit Memo + 재고 반입수익 조정
⚠️ 구축 이슈
회수(Recall) 시나리오에서 Batch Traceability가 결정적입니다. 특정 Batch가 어떤 고객사에 얼마나 납품되었는지 즉시 조회 가능해야 합니다. SAP Batch Where-Used 기능의 설계 우선순위가 높습니다.
💡 불합격 배치 원가는 어떻게 처리하는가? ▼

배치 전량 불합격 판정 시 3가지 선택지가 있습니다:

1. 폐기 (Scrapping): SAP에서 Goods Issue to Scrap. 해당 배치의 누적 원가 전액이 당기 손실(비용)로 인식됩니다.

2. 재작업 (Rework): 별도의 Rework Process Order를 생성. 재작업 비용이 추가로 집계되어 완제품 원가에 반영됩니다. GMP 관점에서 재작업 절차와 일탈 보고(Deviation Report)가 필수입니다.

3. 폐기+보험 청구: 원료 공급사 또는 계약 위반에 따른 품질 이슈라면 공급사 청구. SAP SD의 Debit Memo를 활용합니다.

Section 05

제약 ERP 시스템 생태계 — ERP는 일부일 뿐이다

ISA-95 계층 구조와 ERP·MES·LIMS·QMS의 역할 분담을 그릴 수 있다

📌 핵심 전제
일반 제조업에서는 ERP가 생산 관리의 중심입니다. 제약·바이오에서는 ERP가 여러 전문 시스템 중 하나일 뿐입니다. ERP 혼자 GMP 프로세스를 완결할 수 없습니다.

🏛️ISA-95 자동화 피라미드 — 제약 IT 계층 구조

국제 표준 ISA-95는 제조 IT 시스템을 5개 계층으로 정의합니다. 제약 ERP는 Level 4(사업관리)에 위치하며, Level 2–3(현장)은 MES/SCADA가 담당합니다.

LIMS↔ERP 역가 인터페이스는 제약 ERP 구축의 최고 난이도 과제입니다. 역가 값이 LIMS에서 확정되기 전에 생산을 시작해야 하는 경우, ERP는 표준 역가(Standard Potency)를 사용하고 확정 후 차이를 정산하는 설계가 필요합니다.
💡 MES가 없는 제약사에서 SAP PP-PI만으로 GMP를 충족할 수 있는가? ▼

이론적으로는 가능합니다. SAP PP-PI의 PI Sheet(공정지시서)를 활용하면 ERP 내에서 전자 Batch Record를 부분적으로 구현할 수 있습니다.

그러나 실무적 한계가 있습니다: 실시간 공정 파라미터 자동 수집, 설비별 전자 연동, 복잡한 eBR(전자 배치 기록) 워크플로우는 SAP 표준만으로 구현하기 어렵습니다. 특히 FDA가 요구하는 수준의 공정 데이터 무결성을 PP-PI PI Sheet로 완전히 충족하기 어렵다는 것이 현장 전문가들의 공통적인 의견입니다.

규모와 규제 요건에 따라: 소규모/국내 전용이라면 PP-PI + SAP QM으로 충분할 수 있고, 글로벌 수출 또는 FDA 실사 대상이라면 전용 MES 도입이 필수에 가깝습니다.

Section 06

SAP 구축 핵심 고려사항

PP-PI vs PP 선택 기준, Recipe Management, Batch Classification, Serialization 등 제약 특화 설계 결정사항을 정리한다

🔀PP-PI vs PP — 제약 ERP 첫 번째 설계 결정

제약 고객사를 처음 만나면 반드시 확인해야 하는 질문: "현재 PP를 쓰고 있나요, PP-PI를 쓰고 있나요?"

SAP PP (Production Planning)

이산 제조(Discrete Manufacturing) 중심. BOM + Routing(작업순서)으로 구성. 제약에서는 제형 단순, 소규모, 국내용에서 사용하는 경우 있으나 GMP 문서화 한계.

❌ 제약에서 PP만 사용하면 안 되는 이유
Batch Record (PI Sheet), Master Recipe, 역가 기반 투입량 자동계산 미지원

SAP PP-PI (Production Planning for Process Industries)

화학·제약·식음료 공정 제조 전용. Master Recipe + PI Sheet(Process Instruction Sheet)가 핵심. GMP Batch Record를 전자화하는 데 최적화.

✅ 제약 ERP 표준 선택
역가 BOM, 전자 Batch Record, 공정 파라미터 수집, MES 연동이 모두 PP-PI 기반

📖Master Recipe — 허가 제조법의 ERP 구현

PP-PI의 핵심 마스터 데이터. 규제 허가 제조법이 ERP Master Recipe로 전환됩니다. BOM(재료 목록)과 Routing(공정 순서)을 통합한 구조입니다.

구성 요소내용GMP 의미
Material List역가 기반 원료·부형제 목록허가된 원료 규격 = 법적 근거
Operations혼합→과립→건조→타정→코팅 단계허가된 공정 순서 = 변경 불가
Phases각 공정 단계의 세부 작업PI Sheet로 작업자에게 지시
Process Parameters온도, 시간, 속도 등 공정 조건실측값과 비교 → 일탈 감지
Version 관리레시피 변경 이력 보존허가 변경 없이 버전 변경 = 규제 위반

🏷️Batch Classification — 제약 재고 관리의 핵심

SAP Batch Management + Classification System으로 배치의 모든 특성을 구조화합니다. 제약에서는 다음 특성이 필수입니다:

📅날짜 특성

• 제조일자 (Manufacturing Date)
• 유효기간 (Expiry Date / SLED)
• 검사일 (Test Date)
• 재시험일 (Retest Date)

🧪품질 특성

• 역가 측정값 (Potency %)
• 합격/보류/불합격 상태
• 시험 항목별 결과값
• CoA 번호

🌍허가 특성

• 판매 허가 국가 목록
• 규제기관 코드
• 출하 승인 여부 (Released)
• Quarantine 사유

📍Serialization & Track and Trace — ATTP

한국(의약품 일련번호제), 미국(DSCSA), EU(FMD) 등 주요 시장은 의약품 개별 포장 단위까지 일련번호 관리를 법적으로 의무화합니다.

📌 SAP ATTP (Advanced Track and Trace for Pharmaceuticals)
SAP S/4HANA와 통합되는 Serialization 전용 솔루션. 제조→출하→유통→소비자까지 개별 단위 추적. 기본 SAP SD/MM으로 구현하면 성능 문제 발생 → ATTP별도 설계 검토 필수.
💡 SAP QM 모듈 — 제약에서 필수 활성화 항목 ▼

검사 로트 (Inspection Lot): 모든 입고·생산·출하 시 자동 생성. 시험 완료 전까지 재고 이동 차단.

동적 변경 규칙 (Dynamic Modification): 공급사 품질 이력에 따라 검사 수준 자동 조정 (스킵 검사 포함).

Stability Testing: 보관 조건별 안정성 시험 일정 관리. 유효기간 설정의 과학적 근거.

CAPA 워크플로우: 일탈 발생 → 근본원인 분석 → 시정조치 → 효과 검증까지의 전 과정을 SAP QM 또는 연동 QMS에서 관리.

디지털 서명 (Digital Signature): 검사 결과 승인, 배치 출하 승인 시 적용. 21 CFR Part 11 요건 충족.

Q. 제약 ERP에서 SAP PP-PI를 PP 대신 사용하는 가장 핵심적인 이유는?
① PP-PI가 PP보다 구축 비용이 저렴하기 때문에
② Master Recipe와 PI Sheet를 통해 GMP Batch Record 전자화와 역가 기반 투입량 관리가 가능하기 때문에
③ PP-PI는 SAP S/4HANA에서만 사용 가능하기 때문에
④ PP-PI가 MES와 인터페이스할 때 별도 개발이 불필요하기 때문에
정답: ② — PP-PI의 핵심은 공정 제조(Process Manufacturing) 특화 구조입니다. GMP가 요구하는 전자 Batch Record(PI Sheet)와 레시피 기반 역가 계산이 PP에서는 표준으로 지원되지 않습니다.
Section 07

사업계획(SAC-P) 적용 — 제약 특유의 계획 복잡성

임상 파이프라인 불확실성부터 역가 변동·Clean Room 제약까지, 제약 사업계획의 고유한 설계 요건을 정리한다

🎯제약 사업계획이 다른 LOB보다 어려운 이유

제약 사업계획은 단순한 수요예측이 아닙니다. 규제 허가 일정, 임상 성공률, 역가 변동, 설비 가용성이 모두 계획 변수로 작동합니다.

🔬임상 파이프라인의 불확실성

신약 파이프라인의 각 단계 성공률: Phase 1 (64%) → Phase 2 (35%) → Phase 3 (58%) → 허가 (85%). 이 확률적 변수를 사업계획에 어떻게 반영하는가?

SAC-P 설계 포인트: 허가 시나리오별(낙관/기본/비관) 수요 계획 버전 관리. 허가 지연 시나리오의 P&L 영향 시뮬레이션.

📜허가 일정과 생산계획의 연동

허가 없이는 생산 없습니다. 그런데 허가가 나면 즉시 시장 공급이 필요합니다. Pre-Launch Inventory 전략이 필요하지만 허가 전 상업 생산은 규제상 불가.

SAC-P 설계 포인트: 허가 예상일을 Demand Driver로 설정. 허가 후 Ramp-Up 계획 자동 생성. Launch Stock 시나리오 시뮬레이션.

⚗️역가 변동을 생산계획에 반영하는 법

역가는 배치마다 다릅니다. 사업계획 단계에서는 역가를 정확히 알 수 없으므로, 표준 역가(Standard Potency)를 기준으로 계획하고 실행 시점에 실제 역가를 적용합니다.

생산계획 투입량 계산 (계획 단계)
계획 원료 소요량 = 목표 생산량 × 표준 역가 기준 계수 × (1 / 표준 수율)
※ 실행 시점에 LIMS 실제 역가 반영 → 실제 투입량 = 계획 대비 ±편차 발생
📌 SAC-P 설계 시사점
역가 편차가 큰 바이오의약품의 경우, 표준 역가 대비 ±10~15% 범위의 원료 버퍼를 계획에 반영해야 합니다. SAC-P에서 역가 민감도 분석 시나리오로 구현 가능합니다.

🏭Clean Room 제약과 Capacity Planning

제약 설비는 일반 공장과 다릅니다. Clean Room 등급, 장비 검증(IQ/OQ/PQ) 상태, 세척·소독 시간이 실질적 가용 시간을 크게 줄입니다.

💊약가 및 리베이트 관리

국내 건강보험 약가는 정부가 고시합니다. 또한 병원·도매상에 대한 리베이트(할인)가 실질 수익에 영향. SAC-P 매출 계획 시 공급가 vs 약가 vs 실수익 3단계 구조 설계 필요.

🔄제네릭 출시 시나리오

오리지널 약의 특허 만료 시 제네릭 경쟁사 진입으로 급격한 수요 하락. 특허 만료일을 기준으로 시장 점유율 시나리오 계획이 필수. SAC-P 시나리오 비교 기능 활용.
